Wifi-Basher


Inspiration


One day, when I was 17 years old, I went to my friend's home a laptop that had no ethernet port.
I had no idea what the password was, so I checked the sticker on the router, only to find that the last few characters had faded away.
I went home fustrated that day. A little bit of messing around with netsh wlan in command prompt led me to the export command.
I realized that Windows 10 used .xml files as network profiles to connect to wifi networks.
I combined what little knowledge I had of BATCH files and Python at that time to construct a wifi password brute forcer.

How does it work?

A .bat file has a loop that does the following:

  1. Delete the current network profile
  2. Import the .xml network profile in the same directory
  3. Attempt to connect to the router
  4. Ping google.com and store the result in a .txt file
  5. Run editor.py,which does the following, then store what is printed from the editor.py into another .txt file
    1. Parse the ping .txt result and print if the connection to google.com was a success
    2. Parse the .xml network profile for the location of the password
    3. Modify the password by one character according to a char-set string
  6. Parse the .txt of the result from editor.py and break the loop if a successful connection was made
